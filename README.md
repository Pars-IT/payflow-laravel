# Payflow Laravel

Payflow Laravel is a Laravel 12 payment service for creating payments, dispatching them to a queue, processing them through a payment gateway, and crediting user wallets when payments succeed.

The app includes:

- REST API endpoints for payments, gateways, wallets, health checks, and Mollie webhooks.
- Asynchronous payment processing with Laravel queues and RabbitMQ.
- Redis-backed payment state caching and payment locks.
- Wallet and transaction persistence in MariaDB/MySQL.
- Simulated gateways for local development and a Mollie gateway for hosted checkout flows.
- API documentation generated by Scramble.

## Main Concepts

### Payments

Payments are created through `POST /api/payments`. Each payment starts as `pending` and is processed by `ProcessPaymentJob`.

Payment amounts are stored as integer cents. For example, `1500` means `EUR 15.00`.

### Gateways

Available gateway keys:

- `ideal`: simulated iDEAL gateway. Fails when the amount is lower than `1000`.
- `ing`: simulated gateway that succeeds.
- `abn-amro`: simulated gateway that succeeds for even amounts and fails for odd amounts.
- `mollie`: creates an async Mollie payment and returns a checkout URL after the job has contacted Mollie.

### Wallets

Successful sync payments credit the user's wallet and create a transaction. Mollie payments are finalized through the Mollie webhook at `POST /api/webhooks/mollie`.

### Idempotency

Payment creation requires an `idempotency_key`. Reusing the same key returns the existing payment instead of creating a duplicate.

## API Routes

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/api/health` | Health check. |
| `GET` | `/api/gateways` | List supported gateway keys. |
| `POST` | `/api/payments` | Create a new payment and dispatch processing. |
| `GET` | `/api/payments/{id}` | Get payment status and checkout URL when available. |
| `GET` | `/api/wallets/{userId}` | Get wallet balance for a user. |
| `POST` | `/api/webhooks/mollie` | Receive Mollie payment status callbacks. |
| `GET` | `/docs/api` | Interactive API documentation. |
| `GET` | `/docs/api.json` | OpenAPI JSON document. |

Example payment request:

```bash
curl -X POST http://localhost:8080/api/payments \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": 1,
    "gateway": "ing",
    "amount": 1500,
    "idempotency_key": "demo-payment-1"
  }'
```

Example response:

```json
{
  "id": "payment-uuid",
  "status": "pending"
}
```

Check the payment status:

```bash
curl http://localhost:8080/api/payments/payment-uuid
```

## Local Setup

### Requirements

- Docker and Docker Compose
- Composer

The project does not require Node.js, npm, Vite, or a frontend build step.

### Environment

Create your environment file:

```bash
cp .env.example .env
```

For Docker, make sure these service hosts match the Compose service names:

```dotenv
APP_URL=http://localhost:8080
DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=payment
DB_USERNAME=laravel
DB_PASSWORD=secret
CACHE_STORE=redis
QUEUE_CONNECTION=rabbitmq
REDIS_HOST=redis
RABBITMQ_HOST=rabbitmq
RABBITMQ_PORT=5672
RABBITMQ_USER=guest
RABBITMQ_PASSWORD=guest
RABBITMQ_QUEUE=payments
MAIL_MAILER=log
MOLLIE_API_KEY=test
```

Use a real Mollie API key when testing the Mollie gateway.

### Install Dependencies

```bash
composer install
```

Generate the application key if it has not been set:

```bash
php artisan key:generate
```

### Start Services

```bash
docker compose up -d --build
```

The app is served by Nginx at:

```text
http://localhost:8080
```

RabbitMQ management is available at:

```text
http://localhost:15673
```

Default RabbitMQ credentials are `guest` / `guest`.

### Run Migrations and Seeders

Run migrations inside the app container:

```bash
docker compose exec app php artisan migrate --seed
```

The seeders create demo users and wallets.

### Queue Worker

The `worker` service runs:

```bash
php artisan queue:work rabbitmq --sleep=1 --tries=3 --timeout=90 --backoff=5
```

If you are not using Docker, start a worker manually:

```bash
php artisan queue:work rabbitmq --sleep=1 --tries=3 --timeout=90 --backoff=5
```

## Payment Flow

1. A client creates a payment with `POST /api/payments`.
2. The API stores a pending payment using the idempotency key.
3. `ProcessPaymentJob` is dispatched to RabbitMQ.
4. The job resolves the selected gateway and charges the payment.
5. Sync gateways finalize immediately and credit the wallet.
6. Mollie returns a checkout URL and finalizes later through `POST /api/webhooks/mollie`.
7. Final payment state is cached in Redis and can be read with `GET /api/payments/{id}`.

## Notes

- Keep amounts in cents to avoid floating-point money errors.
- Use unique `idempotency_key` values per payment attempt.
- Mollie webhooks require a reachable `APP_URL`.
- The API docs at `/docs/api` are the best place to inspect current request and response details.
